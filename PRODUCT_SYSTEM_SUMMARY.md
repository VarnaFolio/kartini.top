## 🎯 Как работи системата

### Автоматично генериране на ID

#### Преди:
```
❌ Ръчно броене на продукти
❌ Дублиращи се ID-та
❌ Липса на организация
```

#### Сега:
```
✅ Централен tracking в JSON файл
✅ Ясна система за номериране
✅ Последното ID винаги е видимо
✅ Уникалност гарантирана
```

### Процес на добавяне:

```
1. Провери products_database.json
   └─> "lastProductId": 36

2. Твоето ID = 37
   
3. Копирай шаблон
   └─> PRODUCT_TEMPLATE.html

4. Попълни данни
   ├─> Заглавие
   ├─> Цена
   ├─> Размер
   ├─> Изображение
   └─> Data атрибути

5. Обнови JSON
   ├─> lastProductId: 37
   ├─> Добави в category
   ├─> Добави в products
   └─> Обнови metadata

6. Готово! ✅
```

## 📦 Структура на продукт

### HTML структура:
```html
<article class="product-card">
├─ data-color          (за филтри)
├─ data-style          (за филтри)
└─ data-finish         (за филтри)
   │
   ├─ <div class="product-image">
   │  ├─ <img>         (изображение)
   │  └─ <button>      (бърз преглед)
   │     └─ 7x data-* атрибути
   │
   └─ <div class="product-info">
      ├─ title         (заглавие)
      ├─ size          (размер)
      └─ footer
         ├─ price      (цена)
         └─ actions
            ├─ wishlist (с data-product-id)
            └─ order
```

### JSON структура:
```json
{
  "37": {
    "title": "Име",
    "category": "abstraktni-kartini",
    "price": 89,
    "size": "50x70 см",
    "colors": ["Синьо", "Златисто"],
    "style": ["Абстрактно", "Модерно"],
    "finish": "Гланцово",
    "status": "active"
  }
}
```

## 🚨 Често срещани грешки

### 1. Дублиращи се ID
```
❌ Използване на ID което вече съществува
✅ Винаги провери products_database.json
```

### 2. Грешен формат на цена
```
❌ data-price="89 лв."
✅ data-price="89"
```

### 3. Липсващи data атрибути
```
❌ <article class="product-card">
✅ <article class="product-card" 
           data-color="..."
           data-style="..."
           data-finish="...">
```

### 4. Грешен път към изображение
```
❌ src="images/pic.jpg"        (от pages/)
✅ src="../images/pic.jpg"      (с ../)
```
